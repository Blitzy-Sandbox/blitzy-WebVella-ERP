We decomposed the monolith into 11 independent microservices — 10 .NET 9 Native AOT Lambda functions plus a Node.js JWT authorizer. Each service owns its own database and deploys independently. The serverless model fundamentally changes our cost structure. Instead of paying for always-on servers, we pay only when code executes. Lambda charges per millisecond, DynamoDB on-demand charges per read/write, and S3 costs pennies per gigabyte. When nobody's using the system at 3 AM, the cost is literally zero. Each of the 11 services scales independently — the CRM can handle a spike without over-provisioning invoicing. We've eliminated server management entirely.

The key reliability win: failure isolation. In the monolith, if the email service goes down, the entire ERP is affected. Now, each service is an independent failure domain. Lambda runs across 3 availability zones by default. DynamoDB is multi-AZ replicated. If one service has issues, the other 10 continue operating normally. This eliminates the "whole ERP is down" incident category entirely.

Business Impact: Security

Security improvements are significant and measurable. We've eliminated MD5 password hashing — a known vulnerability — by migrating to AWS Cognito. All secrets are stored in SSM Parameter Store with encryption. JWT validation happens at the API Gateway edge before requests reach any application code. Each Lambda function has the minimum IAM permissions it needs — CRM can't access invoicing data. Lambda functions only exist when invoked, dramatically reducing the attack surface compared to a running server.

Risk Mitigation

We've designed risk mitigation into the architecture. The infrastructure is dual-target — every CDK stack is tested on LocalStack before production. Data migration uses dry-run mode and validation checksums. The event-driven architecture means we can cut over service by service rather than a risky big-bang migration. The 5,338 test suite documents expected behavior comprehensively, reducing the risk of regressions.
